# 別のPCでプロジェクトの続きを行う方法

現在の開発環境（ソースコード）や設定、そしてこのAIアシスタント（Antigravity）との会話を引き継いで、別のPCで同じように作業を続けるための手順をまとめました。

概ね、**「①ソースコードの移動」「②新PCでの環境セットアップ」「③会話履歴（Brain）の引き継ぎ」**の3ステップになります。

---

## 1. 必要なフォルダのコピー（移行元のPCでやること）

このPCから、以下の2つのフォルダをUSBメモリやクラウドストレージ（Google Drive, OneDriveなど）にコピーして、別のPCに持っていきます。

### ① プロジェクトのソースコード
現在開発を行っているメインのフォルダです。
*   **パス**: `C:\tmp\ClientForVE\VisionCameraApp`
*   ※ フォルダサイズを小さくするため、コピーする前にこのフォルダ内にある `node_modules` フォルダと `android` フォルダは削除しておくことを強くおすすめします。（新しいPCで再生成できます）
*   ※ **`.expo` フォルダについて**: `.expo` フォルダは、今お使いのPCのIPアドレスやスマホの接続履歴などを一時的に保存する「キャッシュ（使い捨ての保存領域）」です。コピー先の別のPCで使うと、古いIPアドレスが残ってしまい通信トラブルの原因になることがあるため、**コピーせずに置いていく（削除してからコピーする）のが正解**です。新しいPCで `npx expo start` を実行すると、そのPCの環境に合わせて自動的に新しく作り直されます。

### ② Antigravity の会話履歴・設定フォルダ（任意ですが推奨）
Antigravity（AI）が記憶している現在の会話の文脈、作成したレポート（task.mdやproject_report.mdなど）が入っているフォルダです。これを引き継ぐことで、AIに「あの時の続きをお願い」と言うことができます。
*   **パス**: `C:\Users\nishida\.gemini\antigravity\brain\2444bc14-5d82-4237-9640-4d852c2a641d`
*   ※ `2444bc14-5d82-4237-9640-4d852c2a641d` はこの特定の会話のIDです。

## 2. 新しいPCでの環境セットアップ

新PCでアプリをコンパイル・実行するために、今回行ったのと同じツール群をインストールします。

### ① Node.js のインストール
Chocolatey または Winget（Windows標準）等を使って、Node.jsをインストールします。最も簡単な方法は、コマンドプロンプトを管理者権限で開き、以下を実行することです：
```cmd
winget install OpenJS.NodeJS
```

### ② Expo CLI のインストール
Node.js インストール後、コマンドプロンプトを開き直して以下を実行します。
```cmd
npm install -g eas-cli
```

### ③ Antigravity（今のこのAIエディタ環境）の準備
新しいPCでも、今お使いのAntigravity / VS Code等の環境を同じようにインストール・ログインしてください。

---

## 3. 新しいPCへのデータ復元とプロジェクト再開

### ① フォルダの配置
1.  コピーしてきた **① プロジェクトのソースコード (`VisionCameraApp`)** を、新PCの好きな場所（例：`C:\tmp\` など）に置きます。
2.  コピーしてきた **② Antigravity の会話履歴** を、新PCの同じパスの階層 (`C:\Users\<新PCのユーザー名>\.gemini\antigravity\brain\2444bc14-5d82-4237-9640-4d852c2a641d`) に配置します。これで次回からこの会話履歴が開けるようになります。

### ② プロジェクトの復元（依存関係のインストール）
コマンドプロンプトを開き、配置したソースコードのフォルダに移動して、パッケージを再インストールします。

```cmd
cd C:\配置した場所\VisionCameraApp
npm install
```

### ③ 新しいPCでのアプリ立ち上げ
パッケージのインストールが完了したら、今まで通りプロジェクトを起動し、EASビルドを行います。

```cmd
# EASにログイン（初回のみ）
eas login

# 開発用Androidアプリ(APK)をクラウドでビルド
eas build --profile development --platform android

# 完了したらスマホにAPKをダウンロード・インストールし、開発サーバーを起動
npx expo start
```

## 💡 今後のヒント： さらにスマートに引き継ぐには？

今回はフォルダごとコピーする手順を紹介しましたが、本格的なアプリ開発では **「Git (GitHub)」** というシステムを使うのが一般的です。
次回別のPCで作業する際は、このプロジェクトを GitHub 等にアップロード（プッシュ）しておくと、USBメモリ等を使わずに「新しいPCでダウンロード（プル）して `npm install` するだけ」で一瞬で環境を同期できるようになります。

AIに **「このプロジェクトをGit管理してGitHubに上げる手伝いをして」** と指示を出せば、設定からアップロードまで全てサポート可能です。
